Dokumentasi Pengaturan Environment Variables (PENTING)
1. Pengaturan di Vercel Dashboard (Environment Variables)
• FIREBASE_DATABASE_URL

Letak di Firebase: Buka Firebase Console -> Pilih Proyek Anda -> Pada menu navigasi kiri, pilih Realtime Database.

Cara Ambil: Salin URL teks panjang yang tercantum tepat di bagian atas tabel database Anda (Contoh: https://kreaverse-ai0107-default-rtdb.asia-southeast1.firebasedatabase.app - pastikan salin tanpa tanda garis miring / di bagian akhir URL).

• FIREBASE_DATABASE_SECRET

Letak di Firebase: Klik ikon gerigi (Settings) di sudut kiri atas layar Firebase Console Anda -> Pilih menu Project settings -> Klik tab Service accounts di bagian atas -> Di submenu sebelah kiri tengah, pilih Database secrets.

Cara Ambil: Arahkan kursor Anda ke kolom tabel secret yang disembunyikan, lalu klik tombol Show. Salin token karakter acak yang panjang tersebut ke Vercel sebagai kunci akses aman serverless.

• CRON_SECRET

Cara Buat: Tidak perlu mencari di Firebase. Langsung buat variabel baru di Vercel bernama CRON_SECRET dan isi dengan token pengaman berikut: h4b1StUdI0_Cr0n_SeCrEt_Key_2026.

2. Pengaturan di GitHub Secrets (Settings -> Secrets -> Actions)
• VERCEL_DOMAIN (Penanganan Custom Domain)

Cara Menentukan: Jika website admin Anda masih menggunakan domain bawaan Vercel, masukkan domain vercel Anda (Contoh: kreaverse-ai-kappa.vercel.app).

Apabila menggunakan Custom Domain: Jika Anda sudah menghubungkan domain pribadi (misalnya: admin.kreaverse.ai atau kreaverse.ai), isi variabel VERCEL_DOMAIN ini menggunakan nama domain pribadi Anda tersebut (Contoh: admin.kreaverse.ai - pastikan diisi tanpa menyertakan https:// dan tanpa garis miring / di akhir).

• CRON_SECRET

Cara Atur: Tambahkan secret baru di GitHub bernama CRON_SECRET lalu masukkan nilai yang sama persis seperti di Vercel, yaitu: h4b1StUdI0_Cr0n_SeCrEt_Key_2026. Ini wajib sama agar sistem otentikasi di GitHub Actions diizinkan memicu penyegaran server di backend Vercel Anda.

Panduan Pembuatan & Integrasi Space HF

Ikuti panduan langkah demi langkah di bawah ini untuk membuat server pengunduh (*downloader*) gratis di Hugging Face, mendapatkan kuncinya, dan mengintegrasikannya ke dashboard Kreaverse AI Anda.

Langkah 1: Mendaftar Akun Hugging Face

Buka situs resmi huggingface.co. Klik tombol Sign Up di pojok kanan atas, isi email, buat kata sandi, dan lakukan konfirmasi email Anda hingga akun aktif murni.

Langkah 2: Membuat Wadah Server (New Space)

Klik foto profil bulat Anda di sudut kanan atas halaman Hugging Face -> pilih menu New Space. Atur formulir pembuatannya persis seperti di bawah ini:

  • Owner: Pilih nama akun Anda pribadi.
  • Space Name: Tulis nama wadah server Anda bebas (Contoh: ytdl-downloader).
  • License: Biarkan kosong saja (opsional).
  • Select the Space SDK: Wajib pilih kotak Docker.
  • Docker Template: Pilih kotak Blank (opsi template kosong).
  • Space Hardware: Pilih hardware yang gratisan default yaitu CPU Basic • Free.
  • Public or Private: Pilih Public agar endpoint server bisa dipanggil dari jauh oleh program admin.
  • Scroll ke bawah dan klik tombol Create Space.
Langkah 3: Membuat Access Token Hugging Face

Klik kembali foto profil bulat Anda di kanan atas halaman -> pilih Settings -> Di sidebar sebelah kiri, klik menu Access Tokens -> Klik tombol abu-abu New token di kanan atas, lalu atur:

  • Token Name: Beri nama pengenal bebas (Contoh: HabiAdminPanel).
  • Type: Sangat Wajib memilih opsi Write (Jangan pilih Read! Karena hak akses Write sangat diperlukan agar tombol Rebuild di dashboard admin bisa memerintahkan server melakukan restart jika versi yt-dlp mengalami kerusakan).
  • Klik tombol Generate a token -> Salin kode token panjang yang muncul (Format token selalu diawali dengan teks hf_...). Simpan token ini dengan aman.
Langkah 4: Contoh Pengisian Formulir Dashboard Admin

Untuk menghubungkan server Hugging Face yang telah Anda buat ke dashboard Kreaverse AI ini, silakan isi kolom formulir pendaftaran di atas dengan format berikut:

1. Nama/Label Server:

Beri nama bebas penanda server Anda. Contoh: Server HF Downloader Utama

2. Username Hugging Face:

Isi dengan username akun Hugging Face Anda pribadi. Contoh: kreaverse-dev

3. Space Name (ID Space):

Isi nama wadah Space yang telah Anda buat pada Langkah 2. Contoh: ytdl-downloader

4. URL Direct Space (Web Endpoint):

Gunakan pola penulisan: https://[username]-[nama-space].hf.space. Contoh pengisian: https://kreaverse-dev-ytdl-downloader.hf.space (pastikan tanpa menyertakan tanda garis miring / di bagian akhir).

5. Hugging Face Access Token:

Isi dengan token Write hasil salinan Anda pada Langkah 3. Contoh: hf_QYnSuxxxxxxxxxx...

Panduan Pengaturan DNS / MX Record

Agar domain pribadi Anda bisa menerima pesan secara otomatis di website pengguna, arahkan pengaturan DNS di provider domain Anda ke nilai berikut:

Type: MX
Host/Name: @ (atau kosong)
Value: mx.mail.tm.
Priority: 10

Setelah melakukan perubahan DNS, dibutuhkan waktu sekitar 5 - 30 menit untuk proses propagasi internet sebelum domain siap menerima email [Kejadian Juni 15, 2026].
